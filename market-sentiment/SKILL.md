# 市场情绪判断

## 核心理念

不用死阈值，用**趋势+结构+赚钱效应**三层组合判断。同一个涨停家数（比如46家）在3万亿和1万亿环境下含义完全不同。

## 四层判断框架

### 第一层：宏观水温

| 指标 | 数据源 | 怎么看 |
|------|--------|--------|
| 成交额 | Baostock（sundial 指数数据） | 趋势方向：连续放大→增量行情，连续萎缩→退潮 |
| 涨跌比 | 同花顺 API / sundial dashboard | 指数红但4000家跌→失真，真实水温看涨跌比 |
| 指数位置 | Baostock 日K | 高位滞涨 vs 低位企稳 vs 破位——定大基调 |

### 第二层：短线热度

| 指标 | 数据源 | 判断逻辑 |
|------|--------|----------|
| 涨停家数趋势 | push2ex → sundial `/api/review` | 看趋势不看绝对值：前天129→昨天103→今天46 = 持续萎缩 = 退潮 |
| 封板率 | 同上 | <70% = 资金做多意愿不足 |
| 炸板率 | 同上 | >30% = 分歧加剧 |
| 连板晋级率 | sundial `/api/review` ladder | 连续下滑 = 接力情绪衰退 |
| 连板高度 | 同上 | 被压缩（如4板→3板）= 畏高 |

### 第三层：赚钱/亏钱效应（最关键）

| 指标 | 怎么看 |
|------|--------|
| 高位股表现 | 昨天地天板→今天跌停 = 亏钱效应扩散，教科书退潮 |
| 地天板/核按钮频率 | 核按钮增多 = 恐慌蔓延 |
| 昨日涨停今日溢价 | 昨涨停今低开比例 — 高于30%=退潮 |
| 板块内分化程度 | 龙头涨、跟风跌 = 收缩期；杂毛也涨 = 扩散期 |

**这层没有结构化数据源**，需要从淘股吧热帖、sundial 连板天梯、board-monitor 告警中综合感知。

### 第四层：主线方向

| 指标 | 数据源 | 怎么看 |
|------|--------|--------|
| 有没有清晰主线 | 同花顺热榜（sundial `/api/hotrank`） | 热榜前10集中在1-2个板块=有主线；分散=轮动 |
| 主线是扩散还是收缩 | 板块内涨停家数变化 | 龙头+跟风一起涨=扩散；只涨龙头=收缩 |
| 新旧题材切换 | 热榜概念标签变化 | 新题材能否承接旧题材资金 |

## 五阶段定义

| 阶段 | 特征 | 数据信号 |
|------|------|----------|
| **冰点** | 无主线，高位集体杀跌，涨停稀少 | 涨停<30，封板率<60%，连板高度≤2，涨跌比严重失衡 |
| **修复** | 涨停回升，有板块站出来，出现反包/地天 | 涨停回升至50+，炸板率回落，亏钱效应减弱 |
| **分歧** | 有主线但个股极度分化，板块内高低切 | 涨停40-80间波动，炸板率>25%，主线内部冰火两重天 |
| **高潮** | 涨停爆发，主线全面扩散，杂毛也涨 | 涨停>100，封板率>80%，主线板块集体涨停 |
| **退潮** | 高位崩塌，连板高度连续压缩 | 连板晋级率连续下降，高位龙头炸板/跌停，核按钮频现 |

## 阶段→仓位映射

| 阶段 | 仓位 | 策略 |
|------|:--:|------|
| 冰点 | ≤20% | 空仓或极轻仓试错，等右侧信号 |
| 修复 | 30-50% | 试探性介入率先站出来的板块龙头 |
| 分歧 | 50-70% | 只做核心辨识度，低吸不追高，板块内聚焦龙头 |
| 高潮 | 70-100% | 重仓主线龙头，但要准备随时降仓 |
| 退潮 | ≤30% | 降仓、止盈、不新开仓，防守为主 |

## 数据获取

### 已有接口

```bash
# sundial 复盘（含情绪指标+连板天梯）
curl http://127.0.0.1:8100/api/review

# sundial 热榜
curl "http://127.0.0.1:8100/api/hotrank?date=2026-05-27&slot=15:00"

# board-monitor 状态
curl http://127.0.0.1:8099/status
```

### push2ex 不可用的替代

push2.*.eastmoney.com 全线不通，sundial `/api/review` 的涨停/跌停/炸板/封板率字段会全部返回 0。此时**不能用 sundial 数据做情绪判断**。

替代方案：
- **淘股吧热帖**（`/newIndex/getNowRecommend?pageNo=1` → `/a/{newTopicID}`）：帖子中通常包含涨停家数、封板率、连板高度等结构化复盘数据
- **同花顺热榜 API** 包含 `rise_and_fall` 涨跌幅和概念标签，可间接判断板块热度
- **mootdx** 可获取个股涨停价，但无法批量统计全市场涨停家数

### 🔴 陷阱：sundial review 全零不报错

当 push2ex 不可用时，`/api/review` 返回 HTTP 200 + 全零数据（`up_count: 0, down_count: 0, broken_rate: 0`），不会报错。**这不是「今天没有涨停」而是「数据源挂了」**。看到全零必须先确认数据源状态，不要用零值做情绪判断。

## 实战流程

1. **盘前 5 分钟**：调 sundial `/api/review` 获取昨日情绪数据+连板天梯
2. **盘中**：board-monitor 实时监控，关注炸板率变化和高位股表现
3. **盘后**：sundial 热榜看板块集中度 + 淘股吧热帖感知市场共识
4. **判断阶段** → 匹配仓位策略 → 写入每日复盘计划
